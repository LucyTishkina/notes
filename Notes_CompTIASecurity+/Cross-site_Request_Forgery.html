Cross-site Request Forgery
- one-click attack, session riding
→ XSRF, CSRF (sea surf)
- take advantage of the trust that a web application has for the user
→ the web site trusts your browser
- significant web application development oversight
→ the application should have anti-forgery techniques added
→ usually a cryptographic token to prevent a forgery

CSRF


1) bad guy creates a funds transfer 2) Request is sent as a hyperlink to a user who may already be logged into the bank web site
bad guy------------------------------------------------------------------→ Bank site visitor


3) Visitor clicks the link and unknowingly sends the transfer request to the bank web site
Bank site visitor -------------------------------------------------→ Bank web server


4) Bank validates the transfer and sends the visitor's funds to the bad guy
Bank web server -------------------------------------------→ Bad guy
