Network Address Translation


NAT (Network Address Translation)
- it is estimated that there are over 20 billion devices connected to the Internet (and growing)
→ IPv4 supports around 4.29 billion addresses
- the address space for IPv4 space is exhausted
→ there are no available addresses to assign
- How does it all work?
→ Network Address Translation
- Not the only use of NAT
→ NAT is handy in many situations

Public addresses vs private addresses
RFC 1918 private IPv4 addresses

Classful Description: Class A
- IP address range 10.0.0.0-10.255-255.255
- Number of addresses 16, 777, 216
- Largest CIDR block (subnet mask) 10.0.0.0/8 (255.0.0.0)
- Host ID size 24 bits

Classful Description: 16 contiguous class B's
- IP address range 172.16.0.0 - 172.31.255.255
- Number of addresses 1, 048, 576
- Largest CIDR block (subnet mask) 172.16.0.0/12(255.240.0.0)
- Host ID size 20 bits

Classful Description: 256 contiguous class C's
- IP address range 192.168.0.0 - 192.168.255.255
- Number of addresses 65, 536
- Largest CIDR block (subnet mask) 192.168.0.0/16 (255.255.0.0)
- Host ID size 16 bits

Network Address Translation
- router converts any private addresses for outgoing traffic
- translates back addresses for incoming traffic

NAT overload/PAT (port address translation)
- or source NAT (the source IP is translated)
- used when there are multiple devices

Port forwarding
- 24x7 access to a service hosted internally
→ web server, gaming server, security system, etc.
- External IP/port number maps to an internal IP/port
→ Does not have to be the same port number
- Also Destination NAT or Static NAT
→ destination address is translated from a public IP to a private IP
→ does not expire or timeout
