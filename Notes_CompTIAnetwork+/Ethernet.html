Ethernet


Ethernet frame

Preamble | SFD | Destination MAC | Source MAC | Type | Payload | FCS

Preamble
→ 7 bytes
→ 56 alternating ones and zeros used for synchronization (101010...)

SFD - start frame delimiter
→ 1 byte
→ designates the end of the preamble (10101011)

Destination MAC address
→ 6 bytes

Source MAC address
→ 6 bytes

EtherType
→ 2 bytes
→ Describes the data contained in the payload

Payload
→ 46-1500 bytes
→ Layer 3 and higher data

FCS - frame check sequence
→ 4 bytes
→ CRC checksum of the frame
→ if the sequence does not match then the frame is dropped and the ethernet card keeps track of the number of frames that have been dropped

Ethernet Media Access Control address
- the physical address of a network adapter
- unique to a device

- 48 bits/6 bytes long
- displayed in hexadecimal

- 2 pieces
- OUI (organizationally unique identifier) - the manufacturer
- Network Interface Controller-Specific - the serial number

Duplex
Half-duplex
- a device cannot send and receive data simultaneously
- all LAN hubs are half-duplex
- switch interfaces can be configured as half-duplex
→ but usually only when connecting to other half-duplex devices
- traffic received on one interface is repeated to all other interfaces (ie sent to all devices)
- two devices communicating simultaneously can cause a collision -
→ will wait for a random amount of time and resend communication to hopefully avoid another collision

- CSMA/CD
→ CS - carrier sense
⇒ is there a carrier?
⇒ is there a signal available that we can use to send some data?
→ MA - multiple access
⇒ more than one device on the network
→ CD - collision detect
⇒ collision - two stations talking at once
⇒ identify when data gets garbled
→ Half-duplex ethernet
⇒ no longer used
→ Listen for an opening
→ Send a frame of data
⇒ senf data whenever you can
⇒ there's no queue or prioritization
→ If a collision occurs
⇒ transmit a jam signal to let everyone know a collision has occured -> clears the network
⇒ wait a random amount of time
⇒ retry the transmission

Full-duplex
- data can be sent and received at the same time
- a properly configured switch interface will be set to full-duplex
- intelligently determines the destination

Wireless network
CSMA/CA
→ CA - collision avoidance
⇒ common on wireless networks
→ Collision detection isn't possible
⇒ A sending station can't “hear” other stations
→ Use RTS/CTS
⇒ Ready to send! (I'm ready to send!)
⇒ Clear to send! (You're clear to send!)
→ Solves the “hidden node” problem
⇒ Station A can hear the access point
⇒ Station B can hear the access point
⇒ Station A can't hear station B
