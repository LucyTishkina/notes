TCP and UDP


Operate on OSI Layer 4 (the transport layer)

Allows for multiplexing

TCP - Transmission Control Protocol
- connection-oriented
- formal connection set-up and close
- “reliable” delivery - expect an ack
- recovery from errors
- packets are numbered - can deal with out-of-order messages or retransmissions
- the receiver can manage how much data is set

UDP - User Datagram Protocol

-connectionless
-no formal or open connection
-"unreliable" delivery - no ack
- no error recovery
- no reordering of data or retransmissions
- no flow control - no feedback from the receiver, sender determines rate of data transmission
