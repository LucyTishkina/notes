Ports


- between 0 and 65 535
- port numbers are for communication, not security

Non-ephemerl port numbers (permanent port numbers)
- Ports 0 through 1023
- Usually on a server or service
- not always the case

Ephemeral ports - temporary port numbers
- Ports 1024 through 65,535
- Determined in real-time by the client

TCP and UDP port numbers are not the same - each one gets a set of port numbers



“well known” port numbers
Port 20 and Port 21
FTP - File Transfer Protocol
tcp 20 (active mode data), tcp/21 (control)
- unencrypted file transfer
- authenticates with a username and password
- full-featured (list, add, delete, etc.)


Port 22
tcp/22
- SSH (secure shell)
- encrypted data communication
- login to devices remotely
- console access

- SFTP (secure FTP)
- SSH file transfer protocol
- provides file system functionality
→ resumes interrupted transfers
→ directory listings
→ remote file removal

Port 23
tcp/23
- telnet (telecommunication network)
- login to devices remotely
- console access
- in-the-clear communication
- not best for production systems


Port 25
tcp/25
- SMTP (Simple Mail Transfer Protocol)
- server to server email transfer
- device to server mail transfer
- commonly configured on mobile devices and email clients
- for clients to deal with incoming email - IMAP, POP3


Port 53
udp/53
- DNS (domain name system)
- multiple DNS servers in production
- critical


Port 67 and 68
udp/67, udp/68
- DHCP (Dynamic Host Configuration Protocol)
- Automated configuration of IP address, subnet mask and other options
- requires a DHCP server
→ server, appliance, integrated into a SOHO router, etc.
- dynamic/pooled
→ IP addresses are assigned in real-time from pool
→ each system is given a lease and must renew at set intervals
- DHCP reservation
→ addresses are assigned by MAC address in the DHCP server
→ quickly manage addresses from one location

Port 69
udp/69
TFTP - Trivial File Transfer Protocol
- no authentication
- no encryption
- minimal features - read and write files
- should not be used on production systems

Port 80
tcp/80
- HTTP (hypertext transfer protocol)
- communication in the browser and other applications
- in the clear


Port 110
tcp/110
POP
- Receive emails from an email server
→ Authenticate and transfer
- POP3 (Post Office Protocol version 3)
→ Basic mail transfer functionality

Port 123
udp/123
- NTP - Network Time Protocol
- Switches, routers, firewalls, servers, workstations
→ every device has its own clock
- Time synchronization
→ log files, authentication of information, outage details
- automatic updates
- flexible
→ administrator can set how clocks are updated
- very accurate

Port 143
tcp/143
- Receive emails from an email server
→ Authenticate and transfer

IMAP4 (Internet Message Access Protocol version 4)
- includes management of email inbox from multiple clients


Port 161
udp/161
- SNMP (Simple Network Management Protocol)
- gather statistics from network devices
- v1 - the original
→ structured tables
→ in the clear
- v2
→ bulk transfers
→ in the clear
→ data type enhancements
- v3
→ secure standard
→ message integrity
→ authentication
→ encryption


Port 389
tcp/389
- LDAP (Lightweight Directory Access Protocol)
- Store and retrieve information in a network directory


Port 443
tcp/443
- HTTPS (hypertext transfer protocol secure)
- communication in the browser and other applications
- encrypted


Port 445
tcp/445
- NetBIOS-less
SMB (Server Message Block)
- Protocol used by Microsoft Windows
→ File sharing, printer sharing
→ Also called CIFS (Common Internet File System)


Port 636
tcp/636
- LDAPS (LDAP Secure)
- A non-standard implementation of LDAP over SSL


Port 1720
tcp/1720
- Voice over IP (VoIP) signalling
→ ITU Telecommunication H.32x protocol series
- Setup and manage VoIP sessions
→ call, ring, hang up
- One of the earliest VoIP standards
→ Still in use today


Port 3389
tcp/3389
- RDP (Remote Desk Protocol)
- share a desktop from a remote location
- remote Desktop Services on many Windows versions
- Clients for Windows, MacOS, Linux, etc.
- can connect to an entire desktop or just an application


Port 5060 and 5061
tcp/5060 and tcp/5061
- SIP (Session Initiation Protocol)
- Voice over IP (VoIP) signalling
- Setup and manage VoIP sessions
→ call, ring, hang up
- Extend voice communication
→ Video conferencing
→ Instant messaging
→ File transfer
→ etc
