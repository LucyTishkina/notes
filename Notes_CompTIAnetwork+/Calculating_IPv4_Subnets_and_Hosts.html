Calculating IPv4 Subnets and Hosts


Why subnet the network?
- security
- segmentation
- can't connect every device in the world to every other device


VLSM (Variable Length Subnet Masks)
- Class-based networks are inefficient
→ The subnet mask is based on the network class
- Allow network administrators to define their own masks
→ customize the subnet mask to specific network requirements
- Use different subnet masks in the same classful network
→ 10.0.0.0/8 is the class A network
→ 10.0.1.0/24 and 10.0.8.0/26 would VLSM


Defining subnets
- IP address 10.0.0.0
- Class A, subnet mask 255.0.0.0
- “classful addressing”
- 11111111.00000000.00000000.00000000
- 255.0.0.0
- /8

Can change the subnet mask so that there are more networks and fewer hosts
- 11111111.11111111.11111111.00000000
- 255.255.255.0
- /24
- “classless” addressing

Calculating subnets and hosts
Power of 2
Number of subnets: 2^subnet bits
Hosts per subnet: 2^host bits - 2 [remove broadcast and network address]

IP address 10.1.1.0/24
Network = 8 bits
Subnet = 16 bits
Host = 8 bits
Total subnets = 16 bits = 2^16 = 65536
Hosts per subnet = 8 bits = 2^8 - 2 = 254

IP address 192.168.11.0/26
11111111.11111111.11111111.11000000
Network = 24 bits
Subnet = 2 bits
Host = 6 bits
Total subnets = 2^2 = 4
Hosts per subnet = 2^6 - 2 = 62

IP address 172.16.55.0/21
Network 16 bits
Subnet 5 bits
Host 11 bits
Total subnets = 2^5 = 32
Hosts per subnet = 2^11 - 2 = 2046
